# Template Layout

Use this reference when preparing JSON for `scripts/generate_testcase_workbook.py`.

## Workbook behavior

- `Summary` is rewritten into generation metadata.
- One test-case sheet is created per role.
- `Legend` is preserved from the bundled template.
- Test-case sheets use the `Global Admin` template formatting.

## JSON schema

```json
{
  "jira_key": "SPPAT-1724",
  "jira_url": "https://snpgroup.atlassian.net/browse/SPPAT-1724",
  "story_title": "As a Global Admin ...",
  "platform": "Web CMS",
  "generated_at": "2026-04-25",
  "roles": [
    {
      "role": "Global Admin",
      "sprint": "Sprint 5.1",
      "test_cases": [
        {
          "tc_id": "SPPAT-1724-GA-01",
          "feature_suite": "As a Global Admin ...",
          "platform": "Web CMS",
          "test_case": "Global Admin can search product industry data with a valid keyword",
          "preconditions": [
            "User already logs in as Global Admin",
            "User opens Product Industry page"
          ],
          "steps": [
            "Input a valid keyword with at least 3 characters",
            "Press Enter"
          ],
          "data": [
            "Example keyword: SABUN"
          ],
          "expected": [
            "System filters the list using the entered keyword",
            "Matching data is displayed"
          ],
          "behaviour": "Positive",
          "priority": "High",
          "severity": "Normal",
          "complexity": "Medium",
          "type": "Functional"
        }
      ]
    }
  ]
}
```

## Column mapping

- `NO`: running number per role sheet, generated by script
- `SPRINT`: `roles[].sprint`
- `TC_ID`: `test_cases[].tc_id`
- `LINK STORY`: top-level `jira_url`
- `FEATURE / SUITE`: `test_cases[].feature_suite`, fallback `story_title`
- `ROLE USER`: `roles[].role`
- `PLATFORM`: `test_cases[].platform`, fallback top-level `platform`, fallback `Web CMS`
- `TEST CASE`: `test_cases[].test_case`
- `PRECONDITIONS`: newline-joined `test_cases[].preconditions`
- `STEPS`: newline-joined `test_cases[].steps`
- `DATA`: newline-joined `test_cases[].data`
- `EXPECTED`: newline-joined `test_cases[].expected`
- `BEHAVIOUR`: `test_cases[].behaviour`
- `PRIORITY`: `test_cases[].priority`
- `SEVERITY`: `test_cases[].severity`
- `COMPLEXITY`: `test_cases[].complexity`
- `TYPE`: `test_cases[].type`
- Remaining execution columns: blank

## Defaults

- If `tc_id` is omitted, the script generates `JIRAKEY-ROLECODE-NN`.
- If `feature_suite` is omitted, the script uses `story_title`.
- If `platform` is omitted, the script uses `Web CMS`.
- If `priority` is omitted, the script uses `Medium`.
- If `severity` is omitted, the script uses `Normal`.
- If `complexity` is omitted, the script uses `Medium`.
- If `type` is omitted, the script uses `Functional`.
